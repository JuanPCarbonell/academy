# Exercise 2 — GitHub Setup

**Time: ~15 minutes**

## Goal
Create a GitHub account and set up the repository that will hold your dbt project for the rest of the course. dbt Cloud connects directly to GitHub — every model you write will be version-controlled automatically.

---

## Part A — Create a GitHub Account

1. Go to [github.com](https://github.com) and click **Sign up**.
2. Choose a username (this will be public — use something professional).
3. Verify your email address.

If you already have a GitHub account, skip to Part B.

---

## Part B — Create the Course Repository

1. Once logged in, click the **+** icon (top right) → **New repository**.
2. Fill in:
   - **Repository name:** `dbt-academy`
   - **Visibility:** Private
   - **Initialize with:** tick *Add a README file*
3. Click **Create repository**.

You should now see your empty `dbt-academy` repo at `github.com/<your_username>/dbt-academy`.

---

## Part C — Explore the GitHub Interface

Take 2 minutes to locate:
- The **Code** tab — where your files live
- The **Issues** tab — for tracking work
- The **Pull requests** tab — where code review happens (you'll use this in Exercise 4)
- The green **Code** button — gives you the URL to clone the repo

---

## ✅ Success Criteria

- [ ] GitHub account created and email verified
- [ ] `dbt-academy` repository created (private)
- [ ] You can see the repository at `github.com/<your_username>/dbt-academy`
